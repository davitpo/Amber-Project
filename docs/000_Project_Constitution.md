# 🟠 Amber Project Constitution

## Mission

> Modern Technology. Classic BMW Soul.

The core mission of the Amber Project is to extend the classic BMW motoring experience into the modern era without compromising the elegant, functional soul of the original cabin design. We do not seek to modernize an old BMW, but rather to construct the integrated features BMW engineers might have designed in the late 1990s if modern microcontroller, display, and wireless technology had been available.

Every design decision, interface screen, and mechanical assembly must respect the original vintage vehicle. If an element looks flashy, aftermarket, or out of place on an E36 or similar classic dashboard, it does not belong in this project.

---

## Core Engineering Principles

### 1. OEM First
Every screen layout, font selection, animation, and user interaction must feel like original factory equipment. The behavior must mimic original instrument clusters, onboard computers (OBC), and analog gauges.

### 2. Fully Reversible
No modification requested or required by this project shall need permanent alterations to the vehicle's interior, dashboard, or wiring loom. Any installation must be easily uninstalled to return the car to 100% factory specifications.

### 3. Open Source
Everything we build must be fully open source, transparent, documented, and reproducible by enthusiasts. This ensures independence, longevity, and community ownership.

### 4. Modular
Systems and features should be designed as independent modules. Hardware boards, firmware blocks, and companion software must operate with minimal critical coupling, enabling users to choose only the options they want.

### 5. Offline First
Core functionality—including timekeeping, interface operations, and basic vehicle telemetry—must function completely offline without relying on active internet access, external server APIs, or remote connectivity.

### 6. Quality over Quantity
We prefer a single, flawlessly implemented screen or function over ten half-finished features. We aim for extreme reliability and robustness, matching the high automotive standards of the original vehicles.

### 7. Simplicity
Prefer elegant and simple solutions over overly clever ones. Code should be readable and hardware should be straightforward to build and service.
